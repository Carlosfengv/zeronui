## 样式来源

Zeron UI 的样式入口主要分为组件库和文档站两层：

| 文件 | 职责 |
| --- | --- |
| `packages/ui/src/styles.css` | 组件库主题变量、Tailwind v4 `@theme inline` 映射、自定义状态 variant、共享 utility |
| `apps/docs/app/global.css` | 文档站全局入口、组件库样式导入、Fumadocs 变量桥接、页面布局变量、基础页面样式 |

核心规则：

- 组件库的主题变量以 `packages/ui/src/styles.css` 为准。
- 文档站通过 `@zeron-ui/ui/styles.css` 引入组件库变量。
- 文档站再把 Fumadocs 的 `--color-fd-*` 变量桥接到 Zeron UI 的语义变量，避免出现第二套颜色系统。
- 变量体系基于 Tailwind CSS v4 `@theme inline`，颜色值使用 `oklch()`。

## 加载顺序

`apps/docs/app/global.css` 的导入顺序如下：

```css
@import "tailwindcss";
@import "@zeron-ui/ui/styles.css";
@import "fumadocs-ui/css/neutral.css";
@import "fumadocs-ui/css/preset.css";
@import "tw-animate-css";
```

含义：

- Tailwind 先加载。
- Zeron UI 组件库变量随后注入。
- Fumadocs 基础样式和 preset 再加载。
- `tw-animate-css` 提供动画工具类。

文档站还通过 `@source` 把组件库源码和 MDX 内容纳入 Tailwind 扫描范围：

```css
@source "../../../packages/ui/src/**/*.{js,jsx,ts,tsx}";
@source "./**/*.{js,jsx,ts,tsx,mdx}";
```

## Tailwind 主题映射

`packages/ui/src/styles.css` 使用 `@theme inline` 将 CSS variables 暴露为 Tailwind token。组件中应优先使用语义类名，而不是直接写颜色值。

```tsx
<div className="bg-background text-foreground" />
<button className="bg-primary text-primary-foreground" />
<div className="border-border ring-ring" />
```

### 字体 token

| Tailwind token | CSS variable | 默认值 |
| --- | --- | --- |
| `--font-sans` | `--font-sans` | `Geist, ui-sans-serif, system-ui, sans-serif` |
| `--font-mono` | `--font-mono` | `"Geist Mono", ui-monospace, SFMono-Regular, monospace` |

### 圆角 token

| Tailwind token | 计算方式 | 常见用途 |
| --- | --- | --- |
| `--radius-sm` | `calc(var(--radius) - 4px)` | 小控件、小标签 |
| `--radius-md` | `calc(var(--radius) - 2px)` | 输入框、按钮 |
| `--radius-lg` | `var(--radius)` | 卡片、面板、弹窗 |
| `--radius-xl` | `calc(var(--radius) + 4px)` | 大容器 |

基础值：

```css
:root {
  --radius: 0.625rem;
}
```

## 主题变量

Zeron UI 使用 `:root` 定义默认浅色主题，用 `.dark` 覆盖同名变量实现暗色主题。新增公开变量时，需要同时补齐 `:root`、`.dark` 和 `@theme inline`。

### 基础承载面

| 变量 | 浅色值 | 暗色值 | 含义 |
| --- | --- | --- | --- |
| <ColorIcon value="var(--background)" /> `--background` | `oklch(1 0 0)` | `oklch(0.141 0.005 285.823)` | 页面背景 |
| <ColorIcon value="var(--foreground)" /> `--foreground` | `oklch(0.145 0 0)` | `oklch(0.985 0 0)` | 页面主前景 |
| <ColorIcon value="var(--card)" /> `--card` | `oklch(1 0 0)` | `oklch(0.21 0.006 285.885)` | 卡片背景 |
| <ColorIcon value="var(--card-foreground)" /> `--card-foreground` | `oklch(0.141 0.005 285.823)` | `oklch(0.985 0 0)` | 卡片前景 |
| <ColorIcon value="var(--popover)" /> `--popover` | `oklch(1 0 0)` | `oklch(0.21 0.006 285.885)` | 浮层背景 |
| <ColorIcon value="var(--popover-foreground)" /> `--popover-foreground` | `oklch(0.141 0.005 285.823)` | `oklch(0.985 0 0)` | 浮层前景 |

维护建议：

- `background` 用于大面积页面铺底。
- `card` 和 `popover` 用于从页面背景中抬起内容，不承担品牌强调色职责。
- `foreground`、`card-foreground`、`popover-foreground` 必须分别和对应背景保持可读对比。

## 自定义状态 Variant

`styles.css` 定义了一组 Tailwind custom variant，用来统一组件状态选择器。

| Variant | 匹配条件 | 用途 |
| --- | --- | --- |
| `dark` | `&:is(.dark *)` | 暗色模式 |
| `data-open` | `[data-state="open"]` 或有效的 `[data-open]` | 打开态，如 Popover、Dialog、Dropdown |
| `data-closed` | `[data-state="closed"]` 或有效的 `[data-closed]` | 关闭态 |
| `data-checked` | `[data-state="checked"]` 或有效的 `[data-checked]` | 选中态，如 Checkbox、Switch、Radio |
| `data-disabled` | `[data-disabled="true"]` 或有效的 `[data-disabled]` | 禁用态 |
| `data-active` | `[data-state="active"]` 或有效的 `[data-active]` | 激活态，如 Tabs、导航项 |

示例：

```tsx
<button className="data-active:bg-accent data-disabled:opacity-50" />
```

维护建议：

- Radix 类组件优先使用 `data-state`。
- 自定义组件可以使用布尔属性形式，如 `data-active`、`data-disabled`。
- 避免在每个组件里重复写复杂 attribute selector，统一走这些 variant。

## 动画 token

`@theme inline` 中定义了两个 accordion keyframes：

| Keyframes | 方向 | 高度来源 |
| --- | --- | --- |
| `accordion-down` | 展开 | `--radix-accordion-content-height`，fallback 为 `--accordion-panel-height` |
| `accordion-up` | 收起 | `--radix-accordion-content-height`，fallback 为 `--accordion-panel-height` |

这组动画兼容 Radix Accordion，也允许非 Radix 组件通过 `--accordion-panel-height` 接入。

## 共享 Utility

`styles.css` 定义了 `no-scrollbar`：

```css
@utility no-scrollbar {
  -ms-overflow-style: none;
  scrollbar-width: none;

  &::-webkit-scrollbar {
    display: none;
  }
}
```

适用场景：

- 隐藏滚动条但保留滚动能力。
- 横向 tabs、chip list、移动端横向导航。
- 需要保持界面安静但内容仍可滚动的区域。

## 文档站变量

`apps/docs/app/global.css` 定义了文档站专用布局变量，也负责把 Fumadocs token 桥接到 Zeron UI 主题。

### 布局变量

| 变量 | 值 | 含义 |
| --- | --- | --- |
| `--nav-height` | `61px` | 自定义导航高度 |
| `--fd-nav-height` | `calc(var(--spacing) * 14)` | Fumadocs 导航高度 |
| `--fd-layout-width` | `100%` | 文档布局宽度 |

| `--fd-toc-width` | `240px` | 文档目录宽度 |

页面级覆盖：

- `#nd-page` 使用 `margin-top: calc(var(--nav-height) - 19px)` 修正文档页顶部间距。

## 全局基础样式

`global.css` 在 `@layer base` 中定义基础规则：

```css
* {
  @apply border-border outline-ring/50;
}

body {
  @apply bg-background text-foreground;
}

html {
  scrollbar-gutter: stable;
}
```

补充规则：

- `html, body { overflow-x: clip; }` 用于避免意外横向滚动。
- `#nd-page h1`、`h2`、`h3` 统一应用 `tracking-tight`。
- `figure.shiki` 去掉默认阴影，让代码块视觉更贴近组件库风格。

## 维护建议

- 新增组件颜色时，优先复用现有语义变量，不要在组件内写死 `oklch()`。
- 只有当现有变量无法表达业务语义时，才新增 token。
- 新增 token 时，需要同时补齐 `:root` 和 `.dark`，并在 `@theme inline` 中暴露为 Tailwind token。
- 文档站专用变量应留在 `apps/docs/app/global.css`。
- 组件库通用变量应留在 `packages/ui/src/styles.css`。
- Fumadocs 的 `--color-fd-*` 应继续桥接到 Zeron UI 主题，避免 docs app 漂移成第二套设计系统。
- 当前 `--muted` 行有重复分号 `;;`，不影响浏览器解析，但建议后续清理。

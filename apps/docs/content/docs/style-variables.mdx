## Source files

Zeron UI uses two style entry points:

| File | Responsibility |
| --- | --- |
| `packages/ui/src/styles.css` | Component library theme variables, Tailwind v4 theme mapping, custom variants, shared utilities |
| `apps/docs/app/global.css` | Docs app global imports, Fumadocs token bridge, layout variables, base page styles |

The component library theme source is `packages/ui/src/styles.css`. The docs app imports it through `@zeron-ui/ui/styles.css`, then maps Fumadocs variables back to the same semantic tokens.

## Import order

`apps/docs/app/global.css` loads styles in this order:

```css
@import "tailwindcss";
@import "@zeron-ui/ui/styles.css";
@import "fumadocs-ui/css/neutral.css";
@import "fumadocs-ui/css/preset.css";
@import "tw-animate-css";
```

The docs app also scans component source and MDX content for Tailwind classes:

```css
@source "../../../packages/ui/src/**/*.{js,jsx,ts,tsx}";
@source "./**/*.{js,jsx,ts,tsx,mdx}";
```

## Tailwind theme mapping

`packages/ui/src/styles.css` uses Tailwind CSS v4 `@theme inline` to expose CSS variables as Tailwind tokens.

### Font tokens

| Tailwind token | CSS variable | Default value |
| --- | --- | --- |
| `--font-sans` | `--font-sans` | `Geist, ui-sans-serif, system-ui, sans-serif` |
| `--font-mono` | `--font-mono` | `"Geist Mono", ui-monospace, SFMono-Regular, monospace` |

### Color tokens

| Tailwind token | CSS variable | Meaning |
| --- | --- | --- |
| `--color-background` | `--background` | Default page background |
| `--color-foreground` | `--foreground` | Default text and icon color |
| `--color-card` | `--card` | Card or panel background |
| `--color-card-foreground` | `--card-foreground` | Card or panel foreground |
| `--color-popover` | `--popover` | Floating layer background |
| `--color-popover-foreground` | `--popover-foreground` | Floating layer foreground |
| `--color-primary` | `--primary` | Primary action or emphasis |
| `--color-primary-foreground` | `--primary-foreground` | Foreground on primary surfaces |
| `--color-secondary` | `--secondary` | Secondary action or soft surface |
| `--color-secondary-foreground` | `--secondary-foreground` | Foreground on secondary surfaces |
| `--color-muted` | `--muted` | Muted surface |
| `--color-muted-foreground` | `--muted-foreground` | Secondary text or muted icon color |
| `--color-accent` | `--accent` | Hover, active, menu highlight, selected row |
| `--color-accent-foreground` | `--accent-foreground` | Foreground on accent surfaces |
| `--color-destructive` | `--destructive` | Error, delete, destructive action |
| `--color-border` | `--border` | Default border |
| `--color-input` | `--input` | Form control border or background |
| `--color-ring` | `--ring` | Focus ring |
| `--color-chart-1` to `--color-chart-5` | `--chart-1` to `--chart-5` | Default chart palette |

Use semantic classes in components:

```tsx
<div className="bg-background text-foreground" />
<button className="bg-primary text-primary-foreground" />
<div className="border-border ring-ring" />
```

### Radius tokens

| Tailwind token | Calculation | Typical use |
| --- | --- | --- |
| `--radius-sm` | `calc(var(--radius) - 4px)` | Small controls and labels |
| `--radius-md` | `calc(var(--radius) - 2px)` | Inputs and buttons |
| `--radius-lg` | `var(--radius)` | Cards, panels, dialogs |
| `--radius-xl` | `calc(var(--radius) + 4px)` | Large containers |

Base value:

```css
:root {
  --radius: 0.625rem;
}
```

## Custom variants

`styles.css` defines Tailwind custom variants for common component states.

| Variant | Matches | Use |
| --- | --- | --- |
| `dark` | `&:is(.dark *)` | Dark mode |
| `data-open` | `[data-state="open"]` or active `[data-open]` | Open state |
| `data-closed` | `[data-state="closed"]` or active `[data-closed]` | Closed state |
| `data-checked` | `[data-state="checked"]` or active `[data-checked]` | Checked state |
| `data-disabled` | `[data-disabled="true"]` or active `[data-disabled]` | Disabled state |
| `data-active` | `[data-state="active"]` or active `[data-active]` | Active state |

Example:

```tsx
<button className="data-active:bg-accent data-disabled:opacity-50" />
```

## Animation tokens

The theme defines two accordion keyframes:

| Keyframes | Direction | Height source |
| --- | --- | --- |
| `accordion-down` | Expand | `--radix-accordion-content-height`, fallback `--accordion-panel-height` |
| `accordion-up` | Collapse | `--radix-accordion-content-height`, fallback `--accordion-panel-height` |

This supports Radix Accordion while still allowing non-Radix accordion-like components to opt in through `--accordion-panel-height`.

## Shared utility

`styles.css` defines `no-scrollbar`:

```css
@utility no-scrollbar {
  -ms-overflow-style: none;
  scrollbar-width: none;

  &::-webkit-scrollbar {
    display: none;
  }
}
```

Use it when a region should remain scrollable while hiding the browser scrollbar, such as horizontal tabs or chip lists.

## Base styles

`global.css` applies these base rules:

```css
* {
  @apply border-border outline-ring/50;
}

body {
  @apply bg-background text-foreground;
}

html {
  scrollbar-gutter: stable;
}
```

It also clips horizontal overflow on `html` and `body` to avoid accidental sideways scrolling.

## Maintenance notes

- Add component colors through semantic variables first, not hard-coded `oklch()` values.
- When adding a token, define it in both `:root` and `.dark`.
- Expose new public tokens through `@theme inline`.
- Keep docs-only layout variables in `apps/docs/app/global.css`.
- Keep reusable component-library variables in `packages/ui/src/styles.css`.
- Keep Fumadocs `--color-fd-*` variables bridged to the Zeron UI theme so the docs site does not drift into a second color system.
- The current `--muted` line has a duplicate semicolon. It does not break CSS parsing, but it can be cleaned up later.

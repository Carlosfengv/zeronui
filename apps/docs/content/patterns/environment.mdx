Environment patterns compose `Card`, `Input`, and `Button` primitives into a deploy-settings surface for teams that need to review and edit runtime configuration without leaving context.

## When to Use

Use Environment when people need to scan key/value configuration, reveal individual secrets, and make small edits before deploying.

Prefer this pattern for project settings, deployment dashboards, environment switchers, and admin tools. If the data needs bulk imports, audit history, or permission approvals, pair it with a larger table or workflow.

## Composition

An Environment pattern is built from:

- `Card`: the outer settings surface.
- `CardHeader`, `CardTitle`, and `CardDescription`: environment scope and variable count.
- `Input`: editable variable names and values.
- `Button`: edit, cancel, save, reveal, add, remove, and deploy actions.

## Install

```bash
npx zeron-ui add pattern-environment-variable-editor
```

## Behavior

Keep the default state compact and scannable. Entering edit mode should preserve the last saved values in a draft, let users add or remove rows, and allow canceling without mutating the saved list.

Masked secrets should stay hidden by default and reveal one row at a time. Normalize variable names on save so pasted labels remain deployment-friendly.

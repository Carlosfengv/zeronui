Patterns are business-ready component compositions that sit between primitives and full blocks. Each pattern document describes one pattern component: when to use it, what it is made from, and how to install it.

Use this section for reusable product UI that has more intent than a base component, but is smaller than a full page section. Good examples include page headers, empty states, user menus, data table toolbars, command search, and account switchers.

## Available patterns

- [Item](/patterns/item): item compositions for list rows, menu entries, cards, media rows, and action surfaces.
- [Environment](/patterns/environment): editable deployment variables with masked secret handling.
- [Quota](/patterns/quota): compact usage summaries for billing, plan limits, and account settings.
